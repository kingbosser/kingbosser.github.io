Excel复习之实验篇

Posted on 2016-06-02   |  
  1. 复习方法：实践为主，看书为辅
  2. 复习目标：90分
  3. 复习时间：1天为宜
  4. 复习材料：实验课习题重做，教科书知识点记忆，期末考试前复印店打印资料刷题
  5. 复习策略：总结实验课习题类型，完成操作，归纳要点，完成Excel实践篇的复习

注意：（函数所有知识将在另外一篇文章附上）
实验一：Excel2010基础知识

  1. 理解绝对应用和相对引用（易）
  2. 修改工作表名称：鼠标移至下方表名上—>右键单击—>选择重命名
  3. 修改新建默认工作表个数：点击左上角“文件”—>“选项”—>“常规”里修改包含的工作表
    数即可
  4. 工作表的移动：首先两个工作簿必须打开，鼠标移至下方表名上—>右键单击—>移动或
    复制（再根据提示选择即可）
  5. 选择不连续的单元格：Ctrl+“单击单元格”
  6. 单元格清除与删除的区别：删除是删除单元格本身，其他单元格会填补空缺，影响布局；
    清除不会清除单元格本身，只会清除数据，数据格式，不会影响布局
  7. 拆分、冻结操作：都在“视图”下操作
  8. 设置打印的每一页都有标题行：“页面布局”—>“打印标题”—>进入页面设置的工作表选
    项卡—>选择所打印标题行区域—>确定即可
  9. 在多个窗口中查看数据（水平、垂直、同步滚动）：“视图”—>”全部重排”—>选择并排
    方式—>点击并排查看设置同步滚动
  10. 制作设计同一行中如何换行操作：Ctrl+Enter
  11. 数字文本自定义格式：如输入3，Excel将把它显示为“ZG003”。
    选中区域右键单击—>设置单元格格式—>“数字”选项卡—>自定义—>输入格式控制符”
    ZG”00# 即可。
    Excel自定义格式有四部分组成：
    正数的格式符—>负数的格式符—>0的格式符—>文本的格式符；详见书P77
  12. 下拉菜单输入数据：“数据”—>“数据验证”—>设置—>验证条件为允许序列—>
    选择输入来源
  13. 日期自定义格式：选中区域右键单击—>设置单元格格式—>“数字”选项卡—>日期—>
    选择格式控制符
  14. 货币显示控制：选中区域右键单击—>设置单元格格式—>“数字”选项卡—>货币—>
    选择格式控制符
  15. 控制数字输入范围：“数据”—>“数据验证”—>设置—>验证条件为允许整数—>
    设置数据介于（或其他）—>设置最大、最小值—>确定
  16. 函数if的使用：P110

实验二：数据的输入和格式化

  1. 跨工作表、工作簿引用数据（易）详见P16-P17三维引用
  2. 随机函数：RANDBETWEEN(bottom，top)；rand()是产生大于等于0且小于1的随机数
  3. 随机数的复制保证值不变操作方法：选择仿真数区域—>复制—>粘贴（选择粘贴值）
  4. 设置立体格式表格（上凸或下凹）：利用Ctrl+鼠标隔行选择单元格区域—>先设置左边框
    和上边框为白色—>再次选中设置右下边框为黑色—>将整个区域设置浅灰色背景增强立体视觉
  5. left（），right（），mid（）函数的使用以及&（连接符）的使用
    其中mid（）函数中start从1开始计数
  6. 自定义格式输入文本，如当输入文本时显示“提醒：”+相应的文本内容
    自定义单元格格式为：;;;“提醒”@
  7. 条件格式（色阶、数据条、图标集）: “开始”—>条件格式—>选择所需要表示的条件格式进行设置即可
  8. 插入批注：“审阅”新建批注或者右键点击添加批注

实验三、实验四、实验五：公式及函数的使用

  1. 名称应用：定义一个工作簿级的名称后，该名称就能被工作簿中的各个工作表直接引用
    用指定的方式定义名称：选择区域—>“公式”—>名称管理器中选择根据所选内容创建
    —>在弹出窗口中选择最左列即可
  2. 数组公式的使用：选中结果区域—>输入计算公式—>“Ctrl+Shift+Enter”组合键的使用
  3. Sum()，sumif()，sumifs()函数的使用，详见P113-115
  4. 逻辑函数AND,NOT,OR,TRUE,FALSE的使用，详见P109
  5. Vlookup()精确查找与Vlookup()模糊查找，详见P290
  6. Mod()函数（求余）和Int()函数（取整）的使用
  7. Countif()和countifs()函数的使用
  8. 排名函数RANK()的使用
  9. INDEX()和MATCH()函数的结合使用，详见P286
  10. TEXT()和UPPER()函数的使用
  11. 日期及时间函数的使用，详见P123
  12. 日期函数应用举例：计算工龄、小时加班工资
  13. 文本函数的使用：提取组合信息

实验六：图表处理
17英寸 21英寸 25英寸 29英寸 24英寸
1999年 914 972 1057 1118 1164
2000年 992 1203 1079 1079 1173
2001年 643 1083 1203 1302 1332

  1. 问题一：2000年各类电视机的销售饼图；
    先选中2000年一行—>插入饼图—>图表工具—>设计—>选择数据—>编辑水平分类轴标签—>确定
  2. 问题二：2001年各类电视机的销售折线图，并添加垂直线；
    垂直线添加过程：图表工具选择添加图表元素—>线条—>垂直线
  3. 问题三：1999至2001年的29英寸和34英寸电视的销售折线图，并添加34英寸电视的销售趋势线。
    趋势线填过程：图表工具—>选择添加图表元素—>趋势线—>选择趋势线类型—>选择添加目标—>确定
  4. 双轴图制作
    选择表格区域—>插入—>所有图标中选择组合—>选择系列名称所对应图标格式—>
    在第二个系列名称后次坐标轴选项打勾—>确定即可
  5. 甘特图的制作，难度较大，详见P160
  6. 股市曲线制作，难度较小，直接插入所需图形即可
  7. 常规排序和特殊次序排序
    常规排序：开始—>排序和筛选—>自定义排序—>选择主要关键字，排序依据，次序—>确定
    特殊排序：文件—>选项，弹出Excel选项对话框，单击“高级”—>常规—>编辑自定义列表—>
    输入序列或者选择区域导入—>确定
    然后进行常规排序过程，次序改为自定义序列，选择自己创建的序列排序即可

实验七：数据管理与数据透视表

  1. 统计字符个数LEN()函数
  2. 在表中每一行加一空行：设计排序，先将原表数据按1，3，5……加上序号，
    在表尾序号列添加2，4，6，8……，最后选中所有数据，按序号列升值排列
  3. 数字筛选：选中表头—>点击开始中的排序与筛选—>“筛选”，这时表头项会出现下三角，
    点击下三角，选择数字筛选—>点击不等式—>在弹出的自定义自动筛选方式设置界限值进行筛选
  4. 文本筛选：选中表头—>点击开始中的排序与筛选—>“筛选”，这时表头项会出现下三角，
    点击下三角，选择文本筛选—>勾选需要选择或丢弃的项—>确定

  5. 使用高级筛选提取不重复值：单击任意非空单元格—>数据—>排序与筛选—>选择高级，
    弹出高级选项按钮，列表区域为数据源，条件区域为无，勾选“选择不重复的记录”即可

  6. 筛选多个条件项：理解条件区域中列，行上数据的意义。
    条件“或”：单列上具有多个条件
    条件“与”：同一行具有多个条件，最后利用高级选择即可

  7. 数据透视图，详见P187

“归零”还内心一片宁静

Posted on 2016-05-24   |  

看到题目，也许有人不是很理解，也懒得去理解，没错你此刻很浮躁，需要静一静！

生活在这个时代，难免如此，应该讲这个社会本就是浮躁的。且不论因与果，我们也是
其中一员不是吗？拿我亲身经历以及我身边的人来说事吧，哈哈，我会不会被打！！！

从三年高中拼搏挤进大学的门槛，初入被称作“社会雏形”的大学，依然保持着雄心壮志，
年少的心总是青春飞扬，应聘部门、争当干部、参加科创比赛等等，大学看起来就是那么
充实，令人向往。

转眼，大二开始了。不出预料，那些大一拼命奔波在学生部门，赛事现场的哥们，不是有了
一个令人羡慕的职位，就是拿到了令人眼红的奖项。他们是辅导员以及同伴们欣赏的对象，
往后有很多评奖评优的机会便属于他们，因为这是他们努力的结果，本就是别人应该得的。

而我今天的主题并不在这里，而是发现了一个重大问题，那便是“许多人在模仿别人走路
的时候，发现自己已经找不到路”。

容我解释一番，为什么是模仿？很多人一定会追问，我走自己的路关别人什么事？先不说
我们是跟着党走这么有逼格的话题，举采访几个同学吧。

“同学，你好！请问你对大学学生会怎么想” ——听别人说，学生会有点小黑暗，不过挺锻炼人
“同学，你如何看待四级？” ——学长学姐说，很多公司招聘二四级没过简历就无情被刷
“为什么会有考研的想法” ——辅导员啊，工作的学长学姐，公司要求啊，大学生那么多，又不缺我一个

我们习惯了听着别人的故事生活，久而久之，自己便成了那些故事里的主人公。听着马云、俞敏洪
的创业经历，我便有创业的念头；听学长学姐吹学技术月薪高达能买一辆车，我转而想学技术；听
过来人讲自己，保研要得过奖学习要数一数二，我便像打了鸡血一样，学习科创两不误。所以，
我听的很多人对我最多的疑问：“你为什么那么忙？”，包括我姐也这样讲过我：“感觉你一上了大学
就没空闲过”。

我不否认，这么多人的评价一定有他的道理。对于一个处女座来说，做好每一件事是基本的准则。
大学有太多值得自己去付出的东西，每一件事都有它的价值所在，做了不会错。也有朋友对我讲，
学习和技术你必须舍弃一个，因为人的精力是有限的。而我的回答是，我们的精力的确有限，可是
我们的精力远不止胜任目前所做的一切。不要打着精力有限的幌子说自己不行，那都是别人忽悠你的。

但是，直到有一天，我才发现了我的问题。与其说发现，倒不如说是别人点醒了我。宅腻了寝室，
想换个地方学习，我把目标放在工作室，学姐给我规定的任务一周做完某网站主页，机会难得，我全身心
投入代码的世界，为了那个小小的愿望，从早到晚，除了吃饭睡觉，剩下的时间都是代码的，恰逢
五一节高中同学在重庆聚会，而我却带了电脑过去，干嘛，还能做啥，躲在一个小屋子里敲，持续到凌晨2点，
庆幸的是，我提前2天完成了任务，便急忙向学姐汇报。可是，结果却出乎我意料之外，学姐说你
没注意兼容性，世界崩塌了。解释了蛮久，学姐点评道：“我觉得你很浮躁，需要静下心来学习一段时间”。

从开始的不服气，在朋友的调和下，慢慢放平了心态。我开始反思自己，我回顾自己进大学走过
的点点滴滴，才发现自己真的好久没有静下心学习过了。身兼班长之职，统计信息，组织活动，
做志愿，参加党组织生活，参加各种科创赛事，处理班级问题…

我终于明白，什么叫“瞎忙乎”，试问自己技术学得咋样了，为什么还是一肚子的不自信。连模拟双选会都
不敢去，做了那么多，到底什么才是自己想要的。匆忙赶路的时候，是否忘了目的地在哪里。哲学家说：
“存在即是合理”，事情存在必定有它的价值，如果这样，是不是每件事情都要去做，那肯定完蛋了。

处女座，追求完美！虽顾及不了很多事情，但至少在一件事上始终不放弃，那就是Web前端。
它在我心里不仅是一份混饭吃的工作，也是一项崇高的事业，需要用时间去建立。但是很多时候我选择
忽视了它，把它搁在一边。就向男人总是对女人讲，等我忙完这一阵就抽时间好好陪你，结果都是套路，
都是谎言。等自己真正空闲了，才记起还有一位心爱的人被自己遗忘许久，感情也就不会太强烈了。

所以，让我们学会“归零”吧！回到原点，回到最初，回到刚进校的心态，重新认识“她”，从最初的回眸
到第一次牵手，世界纷繁复杂，只有学会“归零”，时间便会凝固，世界也就安静了…

git命令小记

Posted on 2016-05-23   |  

第一次尝试接触git，有一个很简单的原因。早有耳闻，借助github能搭建个人博客，
恰巧我正苦苦寻觅着一个写技术博客的地方，喜欢简单，实用，能记录自己思想点滴
的地方。经朋友推荐，学习廖雪峰的git教程为主。趁热打铁，简单记一下所学，加
强记忆。（以Windows系统为例）

git资源：廖雪峰官方网站 http://www.liaoxuefeng.com/

git简介：分布式版本控制系统

安装git: 资源下载地址：https://git-for-windows.github.io 下载完成按默认选择按钮安装
设置：本地机器需要指明自己身份（Your Name和email为github官网注册信息）
命令：$ git config –global user.name “Your Name”
$ git config –global user.email “email@example.com”

创建本地仓库（repository）：在合适位置新建目录，进入目录，初始化即可
$ mkdir myrepo //新建目录
$ cd myrepo //进入当前目录
$ pwd //显示当前目录
/Users/michael/learngit
$ git init //初始化，建立空的仓库，供git管理
检测：当前目录多了一个.git目录成功，.git目录默认隐藏，用 $ ls -ah 命令查看

添加文件至仓库：$ git add //可以反复多次使用添加多个文件
$ git commit -m “注释性文字” //提交至版本库

掌握工作区状态：$ git status //查看文件是否修改
$ git diff //查看修改内容

时光机穿梭：$ git reset –hard commit_id //重新回复到commit_id的版本
$ git log //查看历史版本commit_id
$ git reflog //查看未来版本commit_id

撤销修改：（前提必须理解工作区、暂存区、本地版本库的概念）
1.工作区被修改，还未add $ git checkout – file
2.已经add，还未commit $ git reset HEAD file //回到步骤1，在采用步骤1方法完成撤销
3.已经commit，未提交远程库 $ git reset – hard commit_id //版本回退到指定commit_id

删除文件：$ rm file //删除管理文件里的文件，当然也可手动删除
$ git rm file
$ git commit -m “注释性文字”//删除版本库的文件，并提交

远程仓库设置：为了让github远程库识别是本人提交的修改，所以需进行SSH密钥配置
1.创建SSH Key（如果用户主目录下存在.ssh目录，且该目录里面有id_rsa和id_pub文件则跳过步骤1）
$ ssh-keygen -t rsa -C “youremail@example.com” //是在windows下打开git bush
2.登录github官网，打开“Account settings”，“SSH Keys”页面，然后，点“Add SSH Key”，填上任意Title，
在Key文本框里粘贴id_rsa.pub文件的内容，最后add key

添加远程库：1.登陆GitHub，然后，在右上角找到“Create a new repo”按钮，创建一个新的仓库
2.在Repository name填入myrepo(与本地库同名)，其他保持默认设置，点击“Create repository”按钮，
就成功地创建了一个新的Git仓库

将远程库与本地库关联：$ git remote add origin git@server-name:kingbosser/repo-name.git

本地库向远程推送内容：$ git push -u origin master //第一次推送master分支所有内容
$ git push origin master //后续推送master分支内容

SSH警告：第一次使用Git的clone或者push命令连接GitHub时，会得到一个警告

合并分支与冲突解决：
1.两种方式合并：合并分支时，加上–no-ff参数就可以用普通模式合并，合并后的历史有分支，能看出来曾经做过合并，
而fast forward合并就看不出来曾经做过合并。
2.手动解决（查看文件删除错误的，保留正确的，然后再提交）
$ git log –graph //可以查看分支合并图

分支策略：在实际开发中，我们应该按照几个基本原则进行分支管理：
首先，master分支应该是非常稳定的，也就是仅用来发布新版本，平时不能在上面干活；
那在哪干活呢？干活都在dev分支上，也就是说，dev分支是不稳定的，到某个时候，比如1.0版本发布时，
再把dev分支合并到master上，在master分支发布1.0版本；
你和你的小伙伴们每个人都在dev分支上干活，每个人都有自己的分支，时不时地往dev分支上合并就可以了。

bug分支：修复bug时，我们会通过创建新的bug分支进行修复，然后合并，最后删除；
手头任务未完成，需要暂时停下来处理bug，就要存储保护现场，在恢复。
$ git stash //存储工作现场
$ git stash list //查看stash内容列表
$ git stash apply //恢复工作现场
$ git stash drop //删除stash内容
$ git stash pop //恢复并删除stash内容

feature分支：（每添加一个新功能，最好新建一个feature分支）
$ git branch -D //强行删除分支

多人协作工作模式：
首先，可以试图用git push origin branch-name推送自己的修改；
//////////////////////////////////////////////////////////
如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并；
如果合并有冲突，则解决冲突，并在本地提交；
//////////////////////////////////////////////////////////
没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！
//////////////////////////////////////////////////////////
如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，
用命令git branch –set-upstream branch-name origin/branch-name。
//////////////////////////////////////////////////////////
命令行：$ git remote -v //查看远程库信息
$ git push origin branch-name //向远程库推送本地分支
$ git pull //抓取远程新提交
$ git checkout -b branch-name origin/branch-name //在本地创建和远程分支对应的分支
$ git branch –set-upstream branch-name origin/branch-name //建立本地分支和远程分支的关联

标签管理：$ git tag //新建一个标签
$ git tag -a -m “blablabla…” //指定标签信息
$ git tag -s -m “blablabla…” //用PGP签名标签
$ git tag //查看所有标签
$ git push origin //推送一个本地标签
$ git push origin –tags //推送全部未推送过的本地标签
$ git tag -d //删除一个本地标签
$ git push origin :refs/tags/ //删除一个远程标签
